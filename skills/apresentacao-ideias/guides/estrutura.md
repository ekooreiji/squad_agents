# Estrutura de Apresentação

Guia sobre como estruturar apresentações eficazes.

---

## 1. Estrutura Clássica

### 1.1 A Estrutura

```
1. Abertura (Hook)     → Atenção imediata
2. Contexto            → Setup do problema
3. Problema           → O que precisa resolver
4. Solução            → A proposta
5. Benefícios         → Por que funciona
6. Prova             → Evidências
7. Chamada à Ação      → O que fazer depois
```

### 1.2 Timing

| Seção | Tempo |
|-------|-------|
| Hook | 10% |
| Contexto + Problema | 20% |
| Solução + Benefícios | 40% |
| Prova | 20% |
| CTA | 10% |

## 2. Estrutura de Pitch

### 2.1 Pitch Deck Formula

```
Slide 1: Título menarik
Slide 2: Problema (específico)
Slide 3: Solução (única)
Slide 4: Mercado (oportunidade)
Slide 5: Modelo (como ganha)
Slide 6: Tração (progresso)
Slide 7: Equipe (por que vocês)
Slide 8: ask (quanto precisa)
```

### 2.2 Cada Slide = 1 Ideia

```
✓ Uma ideia por slide
✗ Múltiplas ideias = confusão
```

## 3. Estrutura de Presentation

### 3.1 Deck de Projeto

```
Título: Nome do Projeto

1. Background
2. Problema
3. Objetivos
4. Solução proposta
5. Timeline
6. Recursos necessários
7. Próximos passos
8. Perguntas
```

### 3.2 Estrutura Detalhada

```
- Slide de título: O que é
- Slide de contexto: Por que importa
- Slide de escopo: O que inclui
- Slide de design: Como vai ficar
- Slide deTimeline: Quando fica pronto
- Slide de custo: Quanto custa
- Slide de decisão: O que pedir
```

---

## 4. Estrutura de Problema-Solução

### 4.1 Framework

```
PROBLEMA          SOLUÇÃO
[  Situação   ] → [  Proposta   ]
   atual          resultado
                            
[  Dor do    ] → [  Como       ]
   usuário        resolve
```

### 4.2 Exemplo

```
Problema: Usuários abandonam carrinho
Solução: One-click checkout
Resultado: 25% menos abandon
```

---

## 5. Estrutura de Recomenda ção

### 5.1 Para Decisões

```
1. Resumo executivo
2. Opções consideradas
3. Recomendação + rationale
4. Riscos + mitigação
5. Recursos necessários
6. Próximos passos
```

### 5.2 Para Obter Buy-in

```
1. O que precisa ser decidido
2. Por que agora
3. Opções disponíveis
4. Recomendação
5. Impacto da decisão
6. Call to action
```

---

## 6. Transições Entre Slides

### 6.1 Phrases de Transição

| De | Para | Frase |
|----|------|-------|
| Problema | Solução | "Para resolver isso..." |
| Solução | Benefício | "O resultado..." |
| Benefício | Prova | "Prova disso..." |
| Prova | CTA | "Por isso, sugerimos..." |

### 6.2 Evitar

- Saltos sem explicação
- "Próximo slide..."
- "Vamos ver..."
