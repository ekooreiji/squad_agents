# Exemplos de Design Thinking

Exemplos práticos de aplicação de Design Thinking.

---

## Exemplo 1:Checkout Abandonment

### Problema
Usuários abandonam carrinho no checkout.

### Fase 1: Empatia

- Entrevistas com 10 usuários
- Análise de analytics
- Empathy map creado

**Insight**: Usuários não confiam no процессо

### Fase 2: Definir

```
POV: Usuário que compra online precisa
de confiança no checkout porque está
compartilhando dados pessoais.
```

### Fase 3: Idear

- 15 ideias geradas
- 3 priorizadas:
  1.Trust badges visíveis
  2.Checkout em 3 passos
  3.Garantias visible

### Fase 4: Prototipar

- Protótipo de baixa fidelidade em papel
- Teste com 5 usuários

### Fase 5: Testar

- 4/5 users completaram
- Feedback: "Precisam mais trust signals"

### Fase 6: Iterar

- Adicionar mais trust elements
- Novo protótipo
- Novo teste

## Exemplo 2: App de Delivery

### Problema
Usuários não pedem comida por app.

### Fase 1: Empatia

Observação em restaurantes + entrevistas

**Insight**: Usuários mais velhos têm dificuldade com tecnologia.

### Fase 2: Definir

```
HMW: Como podríamos fazer pedido
mais simples para usuários 50+?
```

### Fase 3: Idear

- Botões maiores
- Voz para pedir
- Pedido por telefone
- Versão simplificada

### Fase 4: Prototipar

- Wireframes versões
- Teste com 8 usuários 50+

### Fase 5: Teste

- Botões maiores: 100% sucesso
- Voz: 50% sucesso
- Simplificada: 75% sucesso

### Resultado

- Botões grandes implementados
- Versão simplified lançada

## Exemplo 3: Onboarding

### Problema
Novos usuários não completam cadastro.

### Fase 1: Empatia

- Survey com 200 usuários
- Entrevistas
-Analytics

**Insight**: Formulário muito longo + sem beneficio claro.

### Fase 2: Definir

```
POV: Novo usuário precisa ver valor
rápido do produto porque está
avaliando se vale tempo.
```

### Fase 3: Idear

- Cadastro social media
-Progress indicator
- Tutorial interativo
- Benenefits primeiro

### Fase 4: Prototipar e Testar

- A/B test com 4 versões
- Versão "benefits primeiro" wins

### Resultado

- Conversão up 35%
- Tempo de cadastro down 50%

## Exemplo 4: Service Design

### Problema
Clientes esperam muito no suporte.

### Fase 1: Empatia

- Observação de fila
- Interviews

### Fase 2: Definir

```
POV: Cliente precisa de resposta
rápida porque tempo é dinheiro.
```

### Fase 3: Idear

- Chatbot
- Callback
- Self-service
- FAQ melhorado

### Fase 4: Prototipar

- Role-play scenarios
- Teste com employees

### Fase 5: Teste

- Chatbot: 80% resolução
- Implementado

## Dicas

| Fase | Dica |
|-----|------|
| Empatia | Observar + perguntar |
| Definir | HMW claro |
| Idear | Quantity |
| Prototipar | Rápido e simples |
| Testar | Com usuários reais |
| Iterar | Não takeser attached to idea |
