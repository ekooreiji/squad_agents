# Personas

Guia para criação, validação e utilização de personas no design centrado no usuário.

---

## O que são Personas

Representação fictional de usuários reais baseada em dados de pesquisa, usada para:
- Empoderar decisões de design
- Manter foco no usuário
- Criar empatia com usuários
- Comunicar descobertas de pesquisa

---

## Criação de Personas

### Processo

1. **Coletar dados**: Pesquisas quantitativas e qualitativas
2. **Identificar padrões**: Agrupar usuários por comportamento
3. **Criar perfis**: Desenvolver personas representativas
4. **Validar**: Testar com dados reais
5. **Documentar**: Criar materialize concreto

### Fontes de Dados

| Fonte | Tipo | Uso |
|-------|------|-----|
| Entrevistas | Qualitativo | Profundidade |
| surveys | Quantitativo | Padrões |
| Analytics | Comportamental | Uso real |
| Suporte | Problemas | Frustrações |

## Tipos de Personas

### Primária
Usuário principal do produto. Designar para.

### Secundária
Usuário que afeta decisão do primary.

### Negativa
Usuário que NÃO é o público-alvo.

### Stakeholder
Pessoas internas (dev, suporte, vendas)

---

## Validação de Personas

### Checklist

- [ ] Dados de pesquisa (não suposição)
- [ ] Comportamentos distintos
- [ ] Objetivos claros
- [ ] Frustrações identificadas
- [ ] Nome e foto realistas
- [ ] Citação real included

### Métodos de Validação

1. **Triangulação**: Múltiplas fontes de dados
2. **Revisão por pares**: Outros pesquisadores
3. **Analytics**: Confirmar existência

---

## Utilização

### Em Design

- Referenciar em decisões
- Criar cenários de uso
- Priorizar funcionalidades
- Validar protótipos

### Em Comunicação

- Apresentar para stakeholders
- Alinhar equipe
- Justificar decisões

---

## Boas Práticas

- Não criar baseado em suposição
- Manter atualizados
- Usar dados reais
- Criar máximo 3-4 personas
- Compartilhar com equipe

---

## Armadilhas

- Criar Personas "desejadas"
- Ignorar dados analytics
- Personagens genéricos
- Não atualizar
