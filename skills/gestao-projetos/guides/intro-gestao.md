# Introdução à Gestão de Projetos

Guia introdutório sobre gestão de projetos.

---

## 1. O que é Gestão de Projetos

Gestão de projetos é a aplicação de conhecimentos, habilidades e técnicas para entregar resultados dentro de escopo, prazo e custo definidos.

---

## 2. Papéis em Projetos

### 2.1 Project Manager (PM)

| Responsabilidade | Descrição |
|----------------|-----------|
| Planejar | Criar cronogramas |
| Coordenar | Alinhar equipe |
| Rastrear | Monitorar progresso |
| Comunicar | Relatórios stakeholders |

### 2.2 Product Owner (PO)

| Responsabilidade | Descrição |
|----------------|-----------|
| Definir | Escopo do produto |
| Priorizar | Backlog |
| Validar | Entregas |
| Decidir | Requisitos de negócio |

### 2.3 Team Lead

| Responsabilidade | Descrição |
|----------------|-----------|
| Liderar | Equipe técnica |
| Technicalar | Decisões técnicas |
| Mentorar | Crescimento |
| Entregar | Código/features |

### 2.4 Designer/Developer

| Responsabilidade | Descrição |
|----------------|-----------|
| Criar | Designs/código |
| Entregar | Tasks atribuídas |
| Comunicar | Bloqueios |
| Colaborar | Com equipe |

## 3. Responsabilidades por Papel

### 3.1 Quando Não Há PM

| Alternativa | Quem Assume |
|------------|-------------|
| Coordenação | Team Lead |
| Cronograma | Team Lead + PO |
| Comunicação | Team Lead |

### 3.2 Quando Não Há PO

| Alternativa | Quem Assume |
|------------|-------------|
| Priorização | PM + Team Lead |
| Escopo | Cliente + Team Lead |
| Validação | Cliente |

## 4. Triângulo de Ferro

### 4.1 Conceito

```
    TEMPO
       /\
      /  \
     /    \
    /------\
   ESCOPO  CUSTO
```

### 4.2 Restrições

| Restrição | O que Affecta |
|-----------|-------------|
| Tempo | Prazo, duration |
| Escopo | Features, escopo |
| Custo | Orçamento, recursos |

### 4.3 Trade-offs

| Mudança | Impacto |
|---------|--------|
| + Escopo | + Tempo/Custo |
| - Tempo | + Custo ou - Escopo |
| - Custo | + Tempo ou - Escopo |

---

## 5. Fases do Projeto

### 5.1 Fase de Início (Kickoff)

- Definir escopo
- Identificar stakeholders
- Criar plano inicial
- Alinhar expectativas

### 5.2 Fase de Planejamento

- Detalhar escopo
- Estimar prazos
- Alocar recursos
- Criar cronograma

### 5.3 Fase de Execução

- Produzir entregas
- Rastrear progresso
- Gerenciar mudanças
- Comunicar status

### 5.4 Fase de Encerramento

- Validar entregas
- Conduzir retrospectiva
- Arquivar documentos
- Celebrate成就

---

## 6. Entregáveis por Fase

| Fase | Entregável |
|------|-----------|
| Kickoff | Project Charter |
| Planejamento | Project Plan |
| Execução | Features, releases |
| Encerramento | Retrospectiva |
