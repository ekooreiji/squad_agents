# Brand Architecture

Guia de hierarquia de marca e sistema de sub-marcas.

---

## 1. O que é Brand Architecture

Brand architecture é a estrutura que organiza todas as marcas e sub-marcas de uma empresa.

---

## 2. Modelos de Arquitetura

### 2.1 Monolithic (Branded House)

```
                    EMPRESA
                       │
                       └── Produtos/Serviços
                       
Exemplo: Google (todos usam Google)
```

**Quando usar:**
- Marca forte e reconhecível
- Produtos alinhados com marca principal

### 2.2 House of Brands

```
                   EMPRESA
        ┌────────────┼────────────┐
        ▼            ▼            ▼
     Marca A      Marca B     Marca C
     
Exemplo: P&G (Gillete, Pampers, etc)
```

**Quando usar:**
- Mercados muito diferentes
- Sub-marcas independentes

### 2.3 Endorsed (Hybrid)

```
                    EMPRESA
                       │
        ┌──────────────┼──────────────┐
        ▼             ▼             ▼
    Produto A    Produto B    Produto C
        ✓           ✓             ✓
       [EMPRESA] [EMPRESA]    [EMPRESA]
       
Exemplo: Marriott (com.endorsement)
```

**Quando usar:**
- Beneficiarse da marca principal
- Manter alguma autonomia

### 2.4 Sub-brands

```
                    EMPREendedora
                       │
        ┌──────────────┼──────────────┐
        ▼             ▼             ▼
      Linha A       Linha B      Linha C
      ───────       ───────      ───────
    Produto A1    Produto B1   Produto C1
    Produto A2    Produto B2   
```

## 3. Hierarquia de Marca

### 3.1 Níveis

| Nível | Nome | Descrição |
|-------|------|----------|
| 1 | Corporate | Marca-mãe |
| 2 | Division | Divisões |
| 3 | Product | Produtos |
| 4 | Variant | Versões |

### 3.2 Estrutura Visual

```
┌─────────────────────────────────────────────────────┐
│                   CORPORATE                         │
├─────────────────────────────────────────────────────┤
│  DIVISION 1  │  DIVISION 2  │  DIVISION 3  │
├──────────────┼──────────────┼──────────────┤
│  Product A  │  Product B  │  Product C   │
│  ──────    │  ──────    │  ──────      │
│  Variant 1 │  Variant 1 │  Variant 1  │
└──────────────┴──────────────┴──────────────┘
```

## 4. Naming de Sub-marcas

### 4.1 Convenções

| Modelo | Padrão | Exemplo |
|--------|-------|--------|
| Descriptor | Empresa + tipo | Google Cloud |
| Endorsed | Sub + marca | Marriott Courtyard |
| Standalone | Nome único | Instagram |

### 4.2 Regras

1. Mantenha consistência
2. Documente padrões
3. Defina hierarquia visual

## 5. Estratégia de Endorsement

### 5.1 Types

| Type | Visual | Usage |
|------|--------|-------|
| Explicit | [Logo] by [Parent] | Legal |
| Implicit | Logo only | Quando óbvio |
| None | Standalone | Autonomia total |

### 5.2 Directriz

- Definir quando endorsement é necessário
- Definir onde muncul
- Documentar regras
