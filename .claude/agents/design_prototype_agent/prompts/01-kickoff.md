# Prompt 01 - Kickoff

Prompt para iniciar projeto de prototipação.

---

## Objetivo

Receber e analisar brief inicial do projeto.

---

## Instruções

Ao receber o brief inicial, você deve:

### 1. Identificar Elementos

| Elemento | Extrair |
|----------|---------|
| Projeto | Nome |
| Objetivo | Purpose |
| Usuários | Target audience |
| Funcionalidades | Features needed |
| Plataforma | Web, mobile, tablet |
| Prazo | Timeline |

### 2. Fazer Perguntas Clarificadoras

Se faltarem informações, pergunte:

```
"Para criar o design, preciso saber:
1. [Pergunta 1]
2. [Pergunta 2]
3. [Pergunta 3]"
```

### 3. Estruturar Requirements

Criar um documento com:

```
## Requirements
### Projeto
[Nome]

### Problema
[O que resolve]

### Usuários
[Quem usa]

### Funcionalidades
- [Feature 1]
- [Feature 2]

### Constraints
[Limitações]
```

---

## Exemplo de Uso

### Input do Usuário

```
"Quero criar a página inicial do meu site de cursos online"
```

### Resposta do Agente

```
Entendi! Para criar a homepage, preciso de algumas informações:

1. Qual é o nome do seu curso/negócio?
2. Quem é o público-alvo? (estudantes, profissionais, etc)
3. Já tem marca definida? (cores, logo)
4. Quais são as principais seções que precisa?
5. É priority mobile ou desktop?

Enquanto isso, posso criar uma estrutura inicial baseada em best practices de homepages de cursos online.
```

---

## Checklist de Kickoff

- [ ] Nome do projeto identificado
- [ ] Objetivo claro
- [ ] Usuários definidos
- [ ] Funcionalidades listadas
- [ ] constraints identificadas
- [ ] Perguntas feitas (se necessário)
