# User Research

Guia de pesquisa com usuários para entender necessidades e comportamentos.

---

## 1. Métodos de Pesquisa

### 1.1 Pesquisas Qualitativas

| Método | Quando Usar | Duração | Participantes |
|--------|-----------|---------|------------|
| **Entrevistas** | Profundidade | 30-60 min | 5-10 |
| **Observação** | Contexto real | 1-4h | 5-8 |
| **Focus groups** | Validação | 60-90 min | 6-8 |
| **Diary study** | Uso prolongado | 1-4 semanas | 8-12 |

### 1.2 Pesquisas Quantitativas

| Método | Quando Usar | Amostra |
|--------|-----------|--------|
| **Survey** | Validação ampla | 100+ |
| **Analytics** | Comportamento real | N/A |
| **A/B testing** | Hipóteses | Por teste |

---

## 2. Processo de Pesquisa

### 2.1 Definir Objetivos

```
PASSO 1: Definir perguntas de pesquisa
          │
          ▼
PASSO 2: Selecionar método
          │
          ▼
PASSO 3: Recrutar participantes
          │
          ▼
PASSO 4: Conduzir pesquisa
          │
          ▼
PASSO 5: Analisar dados
          │
          ▼
PASSO 6: Sintetizar descobertas
```

### 2.2 Perguntas de Pesquisa

| Tipo | Exemplos |
|------|----------|
| **Comportamento** | Como os usuários fazem X? |
| **Atitudes** | O que sentem sobre X? |
| **Motivação** | Por que fazem X? |
| **Necessidade** | O que precisam fazer? |

## 3. Criação de Personas

### 3.1 O que são Personas

Personas são representações fictícias de usuários reais baseadas em dados de pesquisa.

### 3.2 Template

```markdown
## Persona: [Nome]

### Informações Básicas
- Idade: [XX]
- Profissão: [Cargo]
- Localização: [Cidade]

### Objetivos
- [Objetivo 1]
- [Objetivo 2]

### Frustrações
- [Frustração 1]
- [Frustração 2]

### Comportamento
- [Dispositivo principal]
- [Frequência de uso]
```
