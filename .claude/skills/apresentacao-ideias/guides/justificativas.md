# Justificar Escolhas de Design

Guia sobre como justificar decisões de design com clareza.

---

## 1. Por que Justificar

### 1.1Importância

| Situação | Por que preciso justificar |
|----------|----------------------|
| Cliente | Quer entender valor |
| Equipe | Precisa de buy-in |
| Stakeholder | Precisa de aprovação |
| Você mesmo | Clareza de pensamento |

### 1.2 Sem Justificativa

- Decisões容易被推翻
- Credibilidade reduzida
- Projeto fica refém de opinião

---

## 2. Framework de Justificativa

### 2.1Estrutura BASE

```
B - Background (Contexto)
A - Alternative (Alternativas consideradas)
S - Selection (Por que escolheu esta)
E - Evidence (Prova de que é a melhor)
```

### 2.2 Aplicação

```
B: "Pesquisa mostra que usuários X"
A: "Consideramos A, B, C"
S: "Escolhemos A porque..."
E: "Testes confirmaram..."
```

---

## 3. Tipos de Justificativa

### 3.1 Baseada em Dados

```
"Justificativa: Dados mostram que 73% dos
usuários preferem A. Pesquisamos e A
tem 35% melhor conversão."
```

### 3.2 Baseada em Pesquisa

```
"Justificativa: Entrevistas com 10
usuários revelaram que X. Design
resolve Y."
```

### 3.3 Baseada em KPIs

```
"Justificativa: Meta de conversão
era 20%. Design A atingiu 28%."
```

### 3.4 Baseada em Boas Práticas

```
"Justificativa: Usamos padrão X
porque Y/Z demonstraram
eficiência em casos similares."
```

---

## 4. Justificativas por Situação

### 4.1 Por que esse Design?

```
"Porque resolve o problema X
do usuário Y da forma Z."
```

### 4.2 Por que não simples?

```
"A simplicidade foi priorizada.
Cada elemento added reduz
conversão em 5% em testes."
```

### 4.3 Por que esse اللون?

```
"Azul porque pesquisas mostram
confiança. Contrast está em
nível WCAG AAA."
```

### 4.4 Por que esse Typography?

```
"Inter porque tem melhor
leibilidade em screens.
Usado por X, Y, Z."
```

---

## 5. Respondendo Críticas

### 5.1 "Eu não curtia"

```
✓ "Entendo. Qual aspect especficamente?"
✓ "Há algum aspect que funciona?"
✓ "Vamos testar com usuários?"
```

### 5.2 "Não segue padrão X"

```
✓ "Priorizamos consistência com Y
   que é mais usado pelo nosso público."
✓ "Podemos adaptar para padrão X."
```

### 5.3 "É muito diferente"

```
✓ "Diferente porque Y. Dados
   mostram que Z."
✓ "Migramos gradualmente
   para reducir impacto."
```

---

## 6. Evidências Aceitas

| Tipo | Exemplo |
|------|--------|
| Analytics | Dados de uso |
| Testes | A/B test results |
| Pesquisa | User interviews |
| Feedback | NPS, reviews |
| Benchmark | Caso similar |
| Heurística | WCAG, guidelines |

## 7. Template de Justificativa

### 7.1 Para Decisão de Design

```
## [Decisão de design]

### Contexto
[Breve description do problema]

### Alternativas consideredas
1. Opção A: [description]
2. Opção B: [description]

### Escolha
[Opção escolhida]

### Justificativa
1. [Razão 1 + evidence]
2. [Razão 2 + evidence]

### Próximos passos
[Teste/Monitor/Review]
```
