# Análise de Tendências

Guia sobre como analisar e validar tendências.

---

## 1. Framework de Análise

### 1.1 MATTER Framework

| Letra | Significado | Pergunta |
|-------|-------------|----------|
| M | Momentum | Está crescendo? |
| A | Access | Todos podem acessar? |
| T | Timing | É o momento certo? |
| T | Test | Há dados de support? |
| E | Evidence | Há proof points? |
| R | Relevance | É relevante para nossos usuários? |

## 3. Matriz de Decisão

### 3.1 Impacto vs Esforço

```
↑ Impacto
│ ★★★★★    │ ●●●●●
│ ★★★★     │ ●●●
│ ★★★      │ ●●
│ ★★       │ ●
├──────────┼────────────→ Esforço
```

### 3.2Priorização

| Quadrante | Ação |
|-----------|------|
| Alto impacto + Bajo esforço | Fazer agora |
| Alto impacto + Alto esforço | Planejar |
| Bajo impacto + Bajo esforço | Eliminar |
| Bajo impacto + Alto esforço | Reconsiderar |

## 4. Análise de Ciclo

### 4.1 Estágios

| Estágio | Identificar | Ação |
|---------|--------------|------|
| Emergindo | Sinais iniciais | Monitorar |
| Crescendo | Aceleração | Testar |
| Pico | Máxima adoção | Avaliar timing |
| Saturando | Crescimento lento | Preparar saída |
| Declinando | Queda | Sair |

## 7. Erros Comum

### 7.1 Evitar

- Achar que tendência = solução
- Copiar sem entender contexto
- Chegar tarde demais

### 7.2 Best Practices

- Validar dados
- Testar pequeno
- Iterar baseado em resultado
