# Importância da Documentação

## Benefícios

| Benefício | Descrição |
|----------|-----------|
| **Reduz Perguntas** | 50% menos perguntas da equipe |
| **Onboarding** | Novos devs produtivos mais rápido |
| **Preserva Conhecimento** | Não depende de uma pessoa |
| **Melhora Qualidade** | Forces a pensar em clareza |

## Custos de Não Documentar

```
┌────────────────────────────────────────────────────────────────┐
│  Sem Documentação (1 ano)                              │
├────────────────────────────────────────────────────────────────┤
│  -3 meses: "Quem sabe como isso funciona?"              │
│  -6 meses: "Por que foi feito assim?"                  │
│  -12 meses: "Ninguém sabe mexer aqui"                 │
└────────────────────────────────────────────────────────────────┘
```

## Quando Documentar

- Todo projeto precisa de README
- Decisões arquiteturais → ADR
- Código complexo → Comments
- API pública → Documentação de API
