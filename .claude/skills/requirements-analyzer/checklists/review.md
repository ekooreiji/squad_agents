# Requirements Review Checklist

Use esta checklist para validar a qualidade de requisitos antes de seguir para implementação.

## Completude

- [ ] Todos os campos do template estão preenchidos
- [ ] Cenário happy path está definido
- [ ] Pelo menos um caso de erro está coberto
- [ ] Casos de borda considerados
- [ ] Pré-condições explicitadas
- [ ] Pós-condições definidas

## Clareza

- [ ] Linguagem é objetiva, sem vaguedades
- [ ] Termos técnicos têm definição clara
- [ ] Não há frases como "normalmente", "geralmente", "rapidamente"
- [ ] Ação do usuário está clara (botão, tela, fluxo)
- [ ] Resultado esperado é verificável

## Consistência

- [ ] Não há conflito com outros requisitos
- [ ] Não contradiz regra de negócio existente
- [ ] Dependências estão identificadas
- [ ] Prioridade está definida e justificada

## Rastreabilidade

- [ ] ID único atribuído
- [ ] Linked to feature/epic superior
- [ ] Origem (fonte) documentada
- [ ] histórico de mudanças registrado

## Testabilidade

- [ ] Requisito é verificável
- [ ] Critérios de aceite são objetivos
- [ ] Pode ser coberto por teste automatizado
- [ ] Tempo de verificação definido (para NFR)

## viabilidade

- [ ] технически possível com recursos atuais
- [ ] Tempo de implementação estimado
- [ ] Dependências externas identificadas

## Segurança (se aplicável)

- [ ] Permissões necessárias definidas
- [ ] Dados sensíveis protegidos
- [ ] audit trail considerado
