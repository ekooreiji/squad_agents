# Necesidades do Cliente

Guia para mapear necessidades explícitas vs implícitas.

---

## 1. O que são Necessidades

Necessidade é o problema ou desejo que o cliente quer atender.

---

## 2. Tipos de Necessidades

### 2.1 Necessidades Explícitas

O que o cliente **dita** claramente.

```
Cliente: "Preciso de um site responsivo."
→ Explícito: site responsivo
```

### 2.2 Necessidades Implícitas

O que o cliente **não diz**, mas está implícito.

```
Cliente: "Quero um site que meus clientes possam usar no celular."
→ Implícito: o cliente percebe que clientes usam mobile
→ Implícito: projeto atual não funciona bem em mobile
```

---

## 3. Framework de Mapeamento

### 3.1 Camadas de Necessidade

| Camada | Descrição | Exemplo |
|-------|-----------|--------|
| Desejo | O que pede | "Quero um site" |
| Necessidade | Problema real | "Meus clientes não me encontram" |
| Medo | O que teme | "Perder clientes para concorrente" |
| Objetivo | O que quer alcançar | "Aumentar vendas 20%" |

### 3.2 Processo

1. Listen to what they say
2. Ask "why" to find the need
3. Ask "what if" to find goals
4. Ask "what if not" to find fears

---

## 4. Técnicas de Descoberta

### 4.1 Perguntas Why

```
Você: "Preciso de um logo melhor."
Pergunta: "Por que você acha que precisa?"
Resposta: "Porque o atual não representa a marca."

→ Por que não representa?
→ O que mudança traria?
```

### 4.2 Perguntas What If

```
Você: "E se o logo resolvesse X?"
→ O que você faria com mais X?"
→ Como isso se conecta com Y?"
```

### 4.3 Perguntas What If Not

```
"E se não tivesse isso?"
→ O que você faria?"
→ Como isso afeta Z?"
```

## 5. Necessidades por Contexto

### 5.1 Novo Projeto

| Tipo | Pergunta-Chave |
|------|----------------|
| Negócio | Qual problema resolve? |
| Usuário | Para quem é? |
| Concorrência | O que existe no mercado? |

### 5.2 Redesenho

| Tipo | Pergunta-Chave |
|------|----------------|
| Atual | O que não funciona? |
| Usuário | O que usuários reclamam? |
| Tecnologia | O que mudou? |

### 5.3 Feature Nova

| Tipo | Pergunta-Chave |
|------|----------------|
| Valor | Para que serve? |
| Integração | Como conecta com resto? |
| Métrica | Como mede sucesso? |

---

## 6. Falácias Comuns

### 6.1 "O Cliente Sabe"

O cliente pode não saber o que precisa.

```
"Ele disse que quer X, mas provavelmente precisa de Y."
```

### 6.2 "Pediu, Logo Precisa"

Às vezes o cliente pede uma solução, não a necessidade.

```
"Quero um app" → pode precisar de acessibilidade
```

### 6.3 "Sempre Fez Assim"

Processos antigos podem não fazer mais sentido.

## 7. Documentando Necessidades

### 7.1 Formato

| Explícito | Implícito | Fonte |
|-----------|-----------|-------|
| Site responsivo | Clientes usam mobile | Entrevista |
| Logo novo | Marca desatualizada | Análise competitiva |

### 7.2 Priorização

| Prioridade | Critério |
|----------|----------|
| Alta | Resolve problema core |
| Média | Melhora experiência |
| Bacha | Nice to have |
