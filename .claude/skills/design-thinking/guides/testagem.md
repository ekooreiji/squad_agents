# Testagem em Design Thinking

Guia sobre como testar com usuários e iterar.

---

## 1. Por que Testar

Testar é donde aprendemos se a solução realmente funciona.

---

## 2. Tipos de Teste

### 2.1 Por Método

| Tipo | Descrição | Quando |
|------|-----------|---------|
| Guerrilla | Público aleatório | Quando precisa dados rápidos |
| Lab | Ambiente controlado | Quando precisa dados precisos |
| Field | Contexto real | Quando precisa contexto |
| Remote | Online | Quando precisa escala |

### 2.2 Por Objetivo

| Objetivo | Tipo de teste |
|---------|--------------|
| Validar conceito | Concept test |
| Validar UX | Usability test |
| Validar produto | Beta test |

---

## 3. Preparação do Teste

### 3.1 Definir Objetivos

```
OBJETIVO: Validar se [solução]
Para [usuário]
Métrica: [como medir sucesso]
```

### 3.2 Criar Tarefas

| Tarefa | O que pedir | Tempo |
|--------|------------|-------|
| 1 | "Complete X" | 2 min |
| 2 | "Encontre Y" | 1 min |
| 3 | " faça Z" | 3 min |

### 3.3 Cenário

```
Cenário: Você está em [situação]
e precisa [objetivo]
```

---

## 4. Durante o Teste

### 4.1 Protocolo

| Fase | Ação |
|------|------|
| Abertura | Bem-vindo, consentimento |
| Contexto | "Pense alto" |
| Tarefas | Observar, gravar |
| Perguntas | Perguntas pós-tarefa |
| Encerramento | Agradecimento |

### 4.2 Perguntas Durante

| Tipo | Exemplo |
|------|--------|
| Pergunta | "O que está pensando?" |
| Encouragement | "Conte mais" |
| Neutra | "O que faria agora?" |

### 4.3 O que EVITAR

- Não ajudar demais
- Não liderar
- Não julgar
- Não dar feedback

---

## 5. After do Teste

### 5.1 Análise

| Métrica | O que medir |
|---------|-------------|
| Taxa de sucesso | % que completou |
| Tempo | Tempo por tarefa |
| Erros | Quantos erros |
| SUS | Score de usability |

### 5.2 Insights

```
"OBSERVAÇÃO: [O que viu]
INSIGHT: [O que isso significa]
IDEIA: [Como iterar]"
```

---

## 6. Iteração

### 6.1 Loop do DT

```
Idear → Prototipar → Testar → Iterar
    ↑__________________↓
```

### 6.2 Decisões After Teste

| Resultado | Ação |
|----------|------|
| Funciona bien | Implementar |
| Parcial | Iterar |
| Não funciona | Voltar a fase anterior |

---

## 7. Errors Comuns

### 7.1 Não Testar Cedo

- Testar tarde = mudanças caras only

### 7.2 Testar Só Interno

- Viés de confirmação

### 7.3 Não Iterar

- Testar sem acting on insights
