# Prototipagem em Design Thinking

Guia sobre como prototipar soluções.

---

## 1. Por que Prototipar

Protótipos permitem aprender rápido, com pouco risco.

---

## 2. Tipos de Protótipo

### 2.1 Por Fidelidade

| Tipo | Detalhe | Uso |
|------|---------|-----|
| Baixa | Papel, wireframe | Testar conceito |
| Média | Mockup, interativo | testar UX |
| Alta | Funcional, produto | Validar mercado |

### 2.2 Por Material

| Tipo | Exemplo | Quando |
|------|---------|--------|
| Analógico | Role-play, Physical | Explorar |
| Visual | Sketches, mockups | Comunicar |
| Digital | Protótipos, clickable | Testar |

---

## 3. Rapid Prototyping

### 3.1 Princípios

- "Fail early, fail cheap"
- Tempo limitado
- Materiais simples
- Foco em aprender

### 3.2 Processo

```
1. Definir objetivo do protótipo
2. Criar em tempo limitado
3. Testar
4. Iterar
```

---

## 4. Protótipos de Serviço

### 4.1 Service Prototype

- Role-play de atendimento
- Storyboard do serviço
- Blueprint de serviço

### 4.2 Exemplo

```
Cenário: Usuário ligando para suporte

Protótipo:
- Uma pessoa = usuário
- Uma pessoa = atendente
- Terceira = observador
```

---

## 5. Protótipos Digitais

### 5.1 Papéis vs Digitais

| Ferramenta | Uso |
|-----------|-----|
| Figma | UI, interativo |
| InVision | Clickable |
| Maze | User testing |
| ProtoPie | Interações |

### 5.2 Quando Usar Digital

- Quando precisa testar interações
- Quando precisa validar com usuários remotos
- Quando precisa dados estruturados

---

## 6. "Make Things" vs "Thing Making"

### 6.1 Thing Making

```
Criar algo físico
- Modelo 3D
- mockup
- Protótipo funcional
```

### 6.2 Make Things

```
Criar representações
- Sketches
- Diagramas
- PoCs
```

---

## 7. Checklist de Protótipo

| # | Item |
|---|------|
| 1 | Objetivo claro |
| 2 | Escopo definido |
| 3 | Tempo limitado |
| 4 | Testável |
| 5 | Feedback possível |
