# Definir o Problema

Guia sobre como definir o problema correctamente.

---

## 1. Por que Definir Bem

Se você define o problema incorrectamente, cualquier solução será incorrecta.

---

## 2. Point of View (POV)

### 2.1 Fórmula

```
[Usuário] precisa [necessidade]
porque [insight]
```

### 2.2 Exemplo

```
Usuário: João, empreendedor de 35 anos
Necessidade: uma forma simples de gerenciar estoque
Porque: ele não tem tempo para processos complexos
```

### 2.3 Como Criar

1. Quem é seu usuário?
2. Qual é a necessidade?
3. Por que isso importa?

---

## 3. Problem Statement

### 3.1Template

```
[Contexto/situação]
[Nome do usuário]:
[Nome do usuário] precisa [necessidade]
em [contexto]
porque [impacto].
```

### 3.2 Exemplo

```
Situação: E-commerce de cosméticos
Problema: Usuários abandonam carrinho na página de checkout

Problem statement:
Loja beauty precisa de maneira de reducir checkout abandonment
porque cada carrinho perdido significa perda de receita.
```

---

## 4. How Might We (HMW)

### 4.1 Fórmula

```
Como podríamos [resolver]
de forma que [benefício]?
```

### 4.2 Exemplos

```
Como podríamos hacer checkout mas rápido
de forma que usuarios no abandonem?

Como podríamos motivar usuarios a completar compra
de forma que conversao increase?

Como podríamos simplificar pagamento
de forma que seja intuitivo?
```

### 4.3 Quantos Criar

| Estágio | Quantidade |
|---------|------------|
| Inicial | 10-15 HMW |
| Selecionar | 3-5 melhores |
| Priorizar | 1-2 principais |

---

## 5. Transformar Insights em Problemas

### 5.1 Insight → Problema

| Insight | Problema |
|---------|----------|
| Usuários não entendem | Melhorar explicação |
| Taxa alta de abandono | Simplificar processo |
| Não encontram produto | Melhorar busca |

### 5.2 Estrutura

```
OBSERVAÇÃO: [O que observou]
INSIGHT: [O que isso significa]
PROBLEMA: [Como traduzir em problema]
```

---

## 6. Priorização

### 6.1 Critérios

| Critério | Pergunta |
|---------|---------|
| Impacto | Quanto afecta usuários? |
| Frequência |多久 acontece? |
| Viabilidade | Dá para resolver? |

### 6.2 Matriz Impacto × Esforço

| | Bajo esforço | Alto esforço |
|---|---|---|
| Alto impacto | Quick wins | Projetos-chave |
| Bajo impacto | BAixo prioridad | BAixo riesgo |

---

## 7. Validação do Problema

### 7.1 Checklist

- [ ] Problema é real?
- [ ] Afeta muitos usuários?
- [ ] Há dados de suporte?
- [ ] Pode ser resolvido?

### 7.2Teste

- Apresentar a outros
- Validar com pesquisas
- Iterar se necessário
