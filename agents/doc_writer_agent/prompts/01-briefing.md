# Prompt 01 - Briefing

Prompt para iniciar projeto de documentação.

---

## Objetivo

Receber e analisar brief inicial do projeto.

---

## Instruções

Ao receber o brief inicial, você deve:

### 1. Identificar Elementos

| Elemento | Extrair |
|----------|---------|
| Projeto | Nome |
| Descrição | O que faz |
| stack | Tecnologias |
| Público | Quem vai ler |
| Estrutura | Arquivos existentes |

### 2. Fazer Perguntas Clarificadoras

Se faltarem informações, pergunte:

```
"Para criar a documentação, preciso saber:
1. [Pergunta 1]
2. [Pergunta 2]
3. [Pergunta 3]"
```

### 3. Estruturar Resumo

Criar um documento com:

```markdown
## Projeto
[Nome]

## Descrição
[O que faz]

## stack
- [Tecnologia 1]
- [Tecnologia 2]

## Público-alvo
[Quem vai ler]

## Tipos de documentação necessária
- [Tipo 1]
- [Tipo 2]
```

---

## Checklist de Briefing

- [ ] Nome do projeto identificado
- [ ] stack tecnológica definida
- [ ] Público-alvo identificado
- [ ] Estrutura do projeto conhecida
- [ ] Tipos de documentação determinados
- [ ] Perguntas feitas (se necessário)
