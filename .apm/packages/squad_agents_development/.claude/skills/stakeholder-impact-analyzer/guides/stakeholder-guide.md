## Matriz de Influência x Interesse

### O Modelo

```
                    Interesse
              Baixo         Alto
          ┌─────────────┬─────────────┐
    Alto  │  MANTER     │  GERENCIAR  │
          │  SATISFEITO │  DE PERTO   │
Influência├─────────────┼─────────────┤
    Baixo │  MONITORAR  │  MANTER     │
          │             │  INFORMADO │
          └─────────────┴─────────────┘
```

### Estratégias por Quadrante

| Quadrante | Estratégia | Ações |
|-----------|-----------|-------|
| **Alta Influência + Alto Interesse** | Gerenciar de perto | Envolver ativamente, reuniões frequentes, incluir em decisões |
| **Alta Influência + Baixo Interesse** | Manter satisfeito | Comunicar proativamente, atender expectativas, não sobrecarregar |
| **Baixa Influência + Alto Interesse** | Manter informado | Atualizar regularmente,征求 input quando relevante |
| **Baixa Influência + Baixo Interesse** | Monitorar | Manter olhos aberto, mínimo esforço de comunicação |

## Análise de Impacto

### Tipo de Impacto

| Tipo | Descrição | Ação |
|------|-----------|------|
| **Positivo** | Beneficia stakeholder | Comunicar benefícios, solicitar apoio |
| **Negativo** | Prejudica stakeholder | Comunicar cedo, mitigar, oferecer suporte |
| **Neutro** | Sem impacto significativo | Manter informado |

### Magnitude

| Magnitude | Descrição | Indicadores |
|-----------|-----------|-------------|
| **Alto** | Impacto significativo | Mudança em trabalho diário, nova habilidade necessária |
| **Médio** | Impacto moderado | Algum ajuste necessário |
| **Baixo** | Impacto mínimo | Alguma cosmetic change |

## Estratégias de Comunicação

### Por Tipo de Impacto

**Impacto Positivo:**
- Comunicar benefícios claramente
- Solicitar apoio/advocacy
- Celebrar bersama

**Impacto Negativo:**
- Comunicar cedo (não esperar)
- Ser honesto sobre desafios
- Oferecer suporte e recursos
- Criar plano de transição

**Impacto Neutro:**
- Manter mínimo overhead
- Informar quando relevante

### Por Magnitude

| Magnitude | Canais | Frequência |
|-----------|--------|-------------|
| **Alto** | Reunião ao vivo, email, documento | Contínua/diária |
| **Médio** | Email, async updates | Semanal |
| **Baixo** | Email, notificação | Pontual |

### Canais de Comunicação

| Canal | Uso | Prós | Contras |
|-------|-----|-----|---------|
| Reunião ao vivo | Decisões importantes | Feedback imediato | Scheduling |
| Email | Comunicação formal | Paper trail | Pode ser ignorado |
| Slack/Teams | Updates rápidos | Informal | Difícil rastrear |
| Documento | Detalhes completos | Referência | Pode não ser lido |
| Demo | Visualização | Entendimento claro | Requer tempo |

## Gestão de Conflitos

### Identificando Objeções

**Stakeholder resiste? Pergunte:**
- "Qual é sua preocupação?"
- "O que precisa para apoiar?"
- "O que mudaria?"

### Tipos de Objeção

| Tipo | Example | Estratégia |
|------|---------|-----------|
| **Processo** | "Isso muda nosso fluxo" | Explicar benefícios, oferecer transição |
| **Recurso** | "Não temos tempo/pessoal" | Oferecer recursos, priorização |
| **Técnico** | "Não é possível" | Validar técnicamente, ajustar escopo |
| **Estratégico** | "Não se alinha com prioritas" | Alinhar com estratégia, demonstrar ROI |

### Estratégias de Mitigação

1. **Antecipar**: Identificar riscos antes
2. **Comunicar cedo**: Não surpreender
3. **Ser empático**: Entender perspectiva
4. **Oferecer suporte**: Recursos para transição
5. **Encontrar aliados**: Unir-se a supporters
6. **Ajustar**: Ser flexível quando possível

## Dependências entre Stakeholders

### Tipos de Dependência

| Tipo | Descrição |
|------|-----------|
| **Informação** | Precisa de info de outro stakeholder |
| **Aprovação** | Precisa de aprovação |
| **Recurso** | Depende de recursos de outro |
| **Calendário** | Depende de cronograma |

### Mapeamento

```
Stakeholder A → Stakeholder B (Tipo de Dependência)
```

Identificar:
- Dependencies que podem bloquear
- Dependencies críticas para sucesso
- Risks de cascata
