# Stakeholder Impact Validation Checklist

Use esta checklist para validar a qualidade da análise de stakeholders.

## Identificação

- [ ] Todos os stakeholders primários identificados
- [ ] Stakeholders secundários considerados
- [ ] Stakeholders indiretos inclusos (muitas vezes esquecidos)
- [ ] Categorias completas: interno, externo, indireto
- [ ] Nenhum stakeholder óbvio deixado de fora

## Categorização de Tipos

### Interno
- [ ] Produto
- [ ] Engenharia/Desenvolvimento
- [ ] Design/UX
- [ ] Quality Assurance
- [ ] Customer Success/Suporte
- [ ] Vendas/Marketing
- [ ] Legal/Compliance
- [ ] Financeiro
- [ ] RH (se aplicável)

### Externo
- [ ] Usuários finais
- [ ] Clientes/Pagadores
- [ ] Parceiros
- [ ] Reguladores
- [ ] Fornecedores

### Indireto
- [ ] Concorrentes
- [ ] Mídia
- [ ] Comunidade

## Análise de Impacto

- [ ] Tipo de impacto definido para cada stakeholder
- [ ] Magnitude justificada e consistente
- [ ] Impactos negativos reconhecidos (não apenas positivos)
- [ ] Impactos neutros considerados

## Matriz de Influência

- [ ] Nível de influência avaliado
- [ ] Nível de interesse avaliado
- [ ] Estratégia adequada para cada quadrante
- [ ] Stakeholders de alta influência/Alto interesse priorizados

## Ações

- [ ] Ações necessárias claras para cada stakeholder
- [ ] Responsável atribuído
- [ ] Prazo definido
- [ ] Canal de comunicação especificado

## Dependências

- [ ] Dependências entre stakeholders mapeadas
- [ ] Dependências críticas identificadas
- [ ] blockers potenciais listados

## Comunicação

- [ ] Plano de comunicação criado
- [ ] Mensagem-chave definida por stakeholder
- [ ] Frequência de comunicação adequada
- [ ] Canais corretos selecionados

## Riscos

- [ ] Riscos identificados
- [ ] Estratégias de mitigação definidas
- [ ] Plano de contingência para objeções

## Validação

- [ ] Revisado com product owner
- [ ] Confirmado com stakeholders afetados
- [ ] Sign-offs necessários identificados
