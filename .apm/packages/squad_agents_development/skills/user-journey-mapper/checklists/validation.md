# User Journey Validation Checklist

Use esta checklist para validar a qualidade do journey map antes definalizar.

## Completude dos Estágios

- [ ] Estágio de Descoberta mapeado
- [ ] Estágio de Consideração mapeado
- [ ] Estágio de Decisão mapeado
- [ ] Estágio de Uso mapeado
- [ ] Estágio de Advocacy mapeado (se aplicável)
- [ ] Transições entre estágios estão claras

## Persona

- [ ] Perfil está detalhado e realista
- [ ] Objetivo principal está claro
- [ ] Motivação está fundamentada
- [ ] Frustrações atuais estão identificadas
- [ ] Canais usados estão mapeados

## Ações e Touchpoints

- [ ] Cada estágio tem pelo menos uma ação
- [ ] Touchpoints são realistas e acessíveis
- [ ] Canais de contato estão atualizados
- [ ] Não há lacunas entre touchpoints

## Análise Emocional

- [ ] Sentimento documentado em cada estágio
- [ ] Variação emocional está explicita (happy to sad)
- [ ] "Highs" e "lows" estão identificados
- [ ] Emoções fazem sentido com a ação

## Pontos de Atrito

- [ ] Pelo menos um pain point identificado por estágio
- [ ] Severidade está classificada (baixa, média, alta)
- [ ] Impacto no usuário está descrito
- [ ] Causa raiz está identificada (se possível)

## Oportunidades

- [ ] Opportunities estão vinculadas aos pain points
- [ ] Impacto está estimado
- [ ] Esforço de implementação estimado
- [ ] Prioridade está classificada
- [ ] Quick wins identificados

## Métricas

- [ ] Métricas relevantes por estágio
- [ ] Dados reais ou estimates Included
- [ ] Metas estão definidas
- [ ] KPIs principais estão claros

## Validação com Dados

- [ ] Dados de analytics corroboram o mapa
- [ ] Feedback de suporte está incluso
- [ ] Entrevistas com usuários realizadas
- [ ] Pesquisas/dados de NPS Included

## Colaboração

- [ ] Multiple stakeholders revisaram
- [ ] Equipe de produto validou
- [ ] Equipe de suporte contribuiu
- [ ] Dados de CS/include
