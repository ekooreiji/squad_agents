# Empatia em Design Thinking

Guia sobre como aplicar empatia para entender usuários.

---

## 1. Por que Empatia

Empatia é o primeiro estágio do DT e fundamental para criar soluções que realmente funcionam.

---

## 2. Técnicas de Pesquisa

### 2.1 Entrevistas

| Fase | Duração | O que fazer |
|------|---------|------------|
| Abertura | 5 min | Rapport, contexto |
| Corpo | 20 min | Perguntas abertas |
| Encerramento | 5 min | Agradecimento |

### 2.2 Perguntas-Chave

| Tipo | Pergunta |
|------|---------|
| Abertura | "Conte sobre a última vez que..." |
| Profundidade | "O que isso significou para você?" |
| Emoção | "Como você se sentiu?" |
| Contexto | "Onde você estava?" |

### 2.3 Observação

| Tipo | O que observar |
|------|----------------|
| Comportamento | O que fazem |
| Contexto | Onde fazem |
| emocionais | Como se sentem |

## 3. Empathy Map

### 3.1 Estrutura

```
+--------------------------------------------------+
|           EMPATHY MAP                            |
+--------------------------------------------------+
|                                                  |
| O QUE PENSA/CRÊ:                                 |
| - [Pensamento 1]                                |
| - [Pensamento 2]                                |
+--------------------------------------------------+
| O QUE VÊ:                                        |
| - [Ambiente 1]                                  |
| - [Ambiente 2]                                  |
+--------------------------------------------------+
| O QUE DIZ/FAZ:                                   |
| - [Ação 1]                                      |
| - [Ação 2]                                      |
+--------------------------------------------------+
| O QUE SENTE/QUER:                                  |
| - [Sentimento 1]                                |
| - [Desejo 1]                                   |
+--------------------------------------------------+
| DORES E FRUSTRAÇÕES:                             |
| - [Problema 1]                                  |
| - [Problema 2]                                  |
+--------------------------------------------------+
```

### 3.2 Como Usar

1. Observar usuário
2. Preencher durante/através
3. Identificar insights
4. Compartilhar com time

## 4. Definição de Personas

### 4.1 Template

| Campo | Descrição |
|-------|-----------|
| Nome | Nome fictício |
| Foto | Imagem representativa |
| Idade | Faixa etária |
| Profissão | O que faz |
| Objetivos | O que quer alcançar |
| Dores | Problemas diários |
| Comportamento | Como usa produto |
| Motivação | O que impulsiona |

### 4.2 Exemplo

```
Nome: João, 35 anos, Empreendedor
Objetivo: Vender mais online
Dores: Sem tempo, visibilidade
Motivação: Crescer negócio
```

## 5. Pesquisas Secundárias

### 5.1 Fontes

- Analytics (dados comportamentais)
- Support tickets (problemas)
- Reviews (opiniões)
- Estudos de mercado

### 5.2 O que Coletar

| Dado | Fonte |
|------|-------|
| Quem | Analytics, surveys |
| Quando | Logs, timing |
| Como | heatmaps, recordings |
| Por quê | Entrevistas |

---

## 6. Erros Comuns

### 6.1 Evitar

| Erro | Por queévitar |
|------|---------------|
| Perguntar "você gostaria" | Tendenciosa |
| Assumir | Sem dados |
| Pesquisar só teammates | Viés interno |
|忽略 contexto | Isolado |

### 6.2 Best Practices

- Multiple fontessim
- Perguntas abertas
- Observar mais, perguntar menos
- Documentar tudo
